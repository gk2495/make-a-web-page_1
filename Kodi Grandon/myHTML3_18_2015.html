HTML Markup, Tags, Attributes, Elements

HTML Markup is made up of Tags. Opening Tags are different from Closing Tags in that Closing Tags have a (/) in front of them. Some examples of tags are <b> = bold and <em> = italic tags.

Tags can have multiple Attributes. An example of a tag that uses an Attribute is the <a> tag = Anchor tag. The <a> tag can use( href=) to create a hyperlink. Another tag that can use an Attribute is the <img> tag. The <img> tag uses (src=) also known as URL.

HTML documents are made of elements. An Element can be looked as a individual component of an HTML document. Click the following link to Review Lesson.

Inline vs Block

Block elements normally start with a new line. Can have height and width "invisable boxes" Examples: <h1>, <p>, and <div>.

Inline elements are normally displayed without line breaks. Examples: <br>, <b>, <a>, <img>, and <span>.

Stage 1: Webpages, Documents, and Structure

HTML, CSS, and DOM

HTML - Hyper Text Markup Language - the standard markup language used to create web pages. HTML controls the structure of a webpage.

CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language. CSS controls the Style of a webpage.

DOM - Document Object Model - a cross- platform and language independent convention for representing and interacting with objects in HTML. The nodes of every document are organized in a tree structure, called the DOM tree.

DIV and SPAN

<Div> - <div> tag defines a division or a section in an HTML document. The <div> tag is used to group block-elements to format them with CSS.

<Span> - The <span> tag is used to group inline-elements in a document. The <span> tag provides no visual change by itself.

CSS

CSS or Cascading Style Sheets is a language used for describing the look and formatting of a document written in a markup language.

Order matters with styles. Styles can be defined in different places and are applied in the following order the default style of a browser, stylesheet in a a seperate file, stylesheet inside HTML, and inline style in an element.

Avoiding Repetition

When writing code you want to avoid repetition. Repetition will increase your likelihood of creating errors while also making your code harder to read. A benefit of avoiding repetition in code would be efficiency in production. Using CSS can assist with avoiding repetition. CSS enables the seperation of document content from document presentation which can reduce complexity and repetition.

The Box Model

Boxes consist of the following:

Content - image or test that appears on the website.

Padding - clears an area around the content effected by the "backround color" of the box.

Border - is inherited by the color property of the box.

Margin - Clears area around the border and is transparent.

Adding Images

You can add an image to a page with the tag <img>. When using an image you should also remember to use the "alt" attribute which displays alternate information to the screen incase the browser cannot display the image. An example is here <img src="img/app.png" alt="This is a screenshot">

Code, Test, Refine

The simple pattern to follow when designing a page.

1. Look for natural boxes.

2. Look for repeated styles and semantic elements.

3. Write your HTML.

4. Apply Styles (from BIGGEST to smallest)

5. Fix things

Stage 2: Telling Computers What To Do

Computer Programming and Computer Science

Computer Programming - is the process fo designing, writing, testing, debugging, and maintaining the source code of computer programs.

Computer Science - is the study of computers and their architecture, languages, and applications, in all aspects, as well as the mathematical structures that relate to computers and computation.

Programming Language

Programming Language is a formal constructed language designed to communicate instructions to a machine, particularly a computer. Programming languages can be used to create programs to control the behavior of a machine or to express algorithms.

Backus-Naur Form

The purpose of Backus Naur Form is to precisely describe exactly the language in a way thats very simple and concise. BNF is often used to describe the syntax of languages used in computing, such as computer programming languages, document formats, instruction sets and communication protocols.

non-terminal -> replacement -> terminal
